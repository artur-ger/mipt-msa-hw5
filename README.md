# mipt-msa-hw5

Репозиторий содержит исходную версию приложения в первом коммите и оптимизированную версию во втором.

## Что было не так в исходной версии

- страница загружалась заново для каждого слова из `words.txt`;
- текст страницы заново разбивался через `split()` для каждого слова;
- в `words.txt` было 7866 строк, но только 93 уникальных слова, поэтому большая часть работы повторялась без пользы.

## Что изменено во втором коммите

- выполняется один HTTP-запрос вместо тысяч повторных запросов;
- частоты слов считаются за один проход по входным данным;
- ответ обрабатывается потоково по чанкам, без хранения и повторного разбиения полного текста;
- при сетевой ошибке приложение завершает работу с понятным сообщением об ошибке вместо длинного traceback.

## Как запустить приложение

```bash
py main.py
```

## Как проверить ускорение и память

```bash
py benchmark.py
```

Бенчмарк сравнивает оптимизированный алгоритм с исходной логикой на одинаковых данных и проверяет, что результат подсчета не изменился.

## Результат проверки

Пример фактического запуска `py benchmark.py` в этом репозитории:

- `speedup=49.66x`
- `memory_reduction=25.15x`

Таким образом, критерии "более чем в 10 раз быстрее" и "более чем в 10 раз меньше памяти" выполняются.
